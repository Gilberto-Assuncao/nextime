# NEXTIME --- WORKFORCE COST & PROFITABILITY INTELLIGENCE

Version: 1.0 Status: Planned Last Updated: 2026-07-21

This future module will derive clearly labeled measures for worked, productive, billable, and non-billable hours. Cost per hour and billed value per hour feed revenue, margin, and profit views by employee, team, company, and project. Reports must distinguish invoiced revenue from calculated profitability and preserve the source period, currency, and calculation version.

Sensitive compensation and profitability data requires explicit permissions, tenant isolation, field-level minimization, auditability, and aggregation rules that prevent unintended disclosure. Employees should not automatically see company margin or peer compensation. Calculations belong in versioned application services, not UI components or database policies.

No profitability calculation or finance workflow is implemented in Sprint 3.5.2.
