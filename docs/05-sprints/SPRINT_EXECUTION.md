# NEXTIME --- SPRINT EXECUTION

Version: 1.0 Status: Active Last Updated: 2026-07-21

# Purpose

This document defines the official execution process for every Sprint in
the NEXTIME project, ensuring disciplined delivery, transparency, and
continuous progress.

------------------------------------------------------------------------

# Daily Workflow

Each workday should follow this sequence:

1.  Review Sprint Goal
2.  Select highest-priority task
3.  Implement the smallest valuable increment
4.  Run tests and validations
5.  Update documentation if needed
6.  Mark task status
7.  Prepare the next task

------------------------------------------------------------------------

# Task Status

Tasks should move through these states:

-   Backlog
-   Ready
-   In Progress
-   In Review
-   Testing
-   Done
-   Blocked

Blocked tasks require a documented reason.

------------------------------------------------------------------------

# Development Rules

-   Follow architecture documentation.
-   Reuse existing components.
-   Keep changes focused.
-   Avoid unrelated refactoring.
-   Maintain TypeScript strict compliance.

------------------------------------------------------------------------

# Code Review

Every completed task should verify:

-   Requirements satisfied
-   Tests passing
-   Documentation updated
-   No regressions introduced
-   Security considerations respected

------------------------------------------------------------------------

# Continuous Integration

Before merging:

-   Lint passes
-   TypeScript passes
-   Automated tests pass
-   Build succeeds

No task is complete if the pipeline fails.

------------------------------------------------------------------------

# Managing Blockers

When blocked:

1.  Document the issue.
2.  Identify dependencies.
3.  Propose alternatives.
4.  Escalate when necessary.

------------------------------------------------------------------------

# Scope Control

During a Sprint:

-   Avoid adding new requirements.
-   Record change requests for future planning.
-   Protect the Sprint Goal.

------------------------------------------------------------------------

# Daily Checklist

-   Sprint Goal reviewed
-   Current task updated
-   Tests executed
-   Documentation synchronized
-   Progress communicated

------------------------------------------------------------------------

# Sprint Completion

A Sprint is complete when:

-   Sprint Goal achieved
-   Accepted tasks delivered
-   Review completed
-   Retrospective completed
-   Remaining work returned to backlog

------------------------------------------------------------------------

# Goal

Provide a repeatable execution process that improves delivery quality,
reduces interruptions, and keeps every Sprint aligned with the NEXTIME
roadmap.
