# NEXTIME --- SPRINT PLANNING

Version: 1.0 Status: Active Last Updated: 2026-07-21

# Purpose

This document defines the official Sprint planning process for NEXTIME,
ensuring predictable delivery, continuous improvement, and alignment
with product goals.

------------------------------------------------------------------------

# Sprint Duration

Standard Sprint length:

-   2 weeks

Exceptions must be approved and documented.

------------------------------------------------------------------------

# Sprint Lifecycle

1.  Backlog refinement
2.  Sprint planning
3.  Development
4.  Code review
5.  Testing
6.  Sprint review
7.  Retrospective

------------------------------------------------------------------------

# Planning Inputs

Each Sprint should include:

-   Product goals
-   Prioritized backlog
-   Technical dependencies
-   Risks
-   Capacity estimate

------------------------------------------------------------------------

# Prioritization

Prioritize work by:

1.  Business value
2.  User impact
3.  Technical dependency
4.  Risk reduction
5.  Technical debt

------------------------------------------------------------------------

# Sprint Goal

Every Sprint must define a single, clear objective that guides
implementation and decision-making.

------------------------------------------------------------------------

# Task Definition

Each task should contain:

-   Description
-   Acceptance criteria
-   Dependencies
-   Estimated effort
-   Responsible owner

------------------------------------------------------------------------

# Acceptance Criteria

Work is complete only when:

-   Requirements implemented
-   Tests pass
-   Documentation updated
-   Code reviewed
-   Definition of Done satisfied

------------------------------------------------------------------------

# Sprint Review

Review:

-   Completed work
-   Demonstrations
-   Remaining backlog
-   Metrics
-   Stakeholder feedback

------------------------------------------------------------------------

# Retrospective

Discuss:

-   What went well
-   What can improve
-   Action items
-   Process changes

Track actions into the next Sprint.

------------------------------------------------------------------------

# Metrics

Monitor:

-   Velocity
-   Completion rate
-   Defects
-   Lead time
-   Cycle time

------------------------------------------------------------------------

# Goal

Provide a repeatable Sprint planning process that supports predictable
delivery and continuous improvement.
