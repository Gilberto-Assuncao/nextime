# NEXTIME --- CHANGELOG POLICY

Version: 1.0 Status: Active Last Updated: 2026-07-21

# Purpose

Define the official policy for recording changes made to the NEXTIME
project.

A well-maintained changelog improves traceability, communication and
release management.

------------------------------------------------------------------------

# Principles

-   Every significant change must be recorded.
-   Entries should be concise and factual.
-   The changelog reflects released or approved changes, not work in
    progress.

------------------------------------------------------------------------

# Changelog Format

Use the following sections when applicable:

## Added

New features and capabilities.

## Changed

Behavior changes or improvements.

## Fixed

Bug fixes.

## Removed

Deprecated or deleted functionality.

## Security

Security-related improvements and fixes.

------------------------------------------------------------------------

# Versioning

Follow Semantic Versioning:

-   MAJOR.MINOR.PATCH

Examples:

-   1.0.0
-   1.2.0
-   1.2.3

------------------------------------------------------------------------

# Writing Guidelines

Each entry should:

-   Describe what changed.
-   Explain the impact when necessary.
-   Avoid implementation details.
-   Reference the related Sprint when appropriate.

Example:

-   Added employee availability management.
-   Fixed timezone conversion in timesheets.
-   Improved dashboard loading performance.

------------------------------------------------------------------------

# Responsibilities

The changelog should be updated whenever:

-   A Sprint is completed.
-   A release is prepared.
-   A significant architectural change is approved.

------------------------------------------------------------------------

# Review

Before publishing:

-   Verify technical accuracy.
-   Remove duplicate entries.
-   Keep chronological order.
-   Confirm version number.

------------------------------------------------------------------------

# Goal

Maintain a reliable historical record of NEXTIME's evolution for
developers, reviewers and future maintenance.
