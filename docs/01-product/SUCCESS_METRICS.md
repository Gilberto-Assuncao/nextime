# NEXTIME --- SUCCESS METRICS

Version: 1.0 Status: Active Last Updated: 2026-07-21

# Purpose

This document defines the key metrics used to evaluate the success of
the NEXTIME platform.

The objective is to align product, engineering and business teams around
measurable outcomes.

------------------------------------------------------------------------

# Product KPIs

Track:

-   Monthly Active Companies (MAC)
-   Monthly Active Users (MAU)
-   Daily Active Users (DAU)
-   Time Entries per Day
-   Projects Created
-   Timesheets Submitted
-   Timesheets Approved

Review frequency: Monthly.

------------------------------------------------------------------------

# Business KPIs

Monitor:

-   Monthly Recurring Revenue (MRR)
-   Annual Recurring Revenue (ARR)
-   Customer Acquisition Cost (CAC)
-   Customer Lifetime Value (LTV)
-   Churn Rate
-   Net Revenue Retention (NRR)

Review frequency: Monthly and Quarterly.

------------------------------------------------------------------------

# Technical KPIs

Track:

-   API response time
-   Error rate
-   Availability (uptime)
-   Database latency
-   Deployment success rate
-   Mean Time to Recovery (MTTR)

Review frequency: Weekly.

------------------------------------------------------------------------

# Adoption Metrics

Measure:

-   New tenant creation
-   User invitations accepted
-   Feature adoption rate
-   Time to first value
-   Daily usage frequency

------------------------------------------------------------------------

# Customer Satisfaction

Indicators:

-   Net Promoter Score (NPS)
-   Customer Satisfaction Score (CSAT)
-   Support response time
-   Support resolution time

Review feedback continuously.

------------------------------------------------------------------------

# Quality Metrics

Monitor:

-   Production defects
-   Escaped bugs
-   Test coverage
-   Failed deployments
-   Security incidents

------------------------------------------------------------------------

# Success Targets

Initial goals:

-   Stable production environment
-   High user retention
-   Low critical defect rate
-   Consistent platform availability
-   Positive customer feedback

Targets should evolve as the platform matures.

------------------------------------------------------------------------

# Review Process

Metrics should be reviewed:

-   Weekly by engineering
-   Monthly by product
-   Quarterly by leadership

Adjust priorities based on measurable outcomes.

------------------------------------------------------------------------

# Goal

Provide objective measurements that guide continuous improvement and
ensure NEXTIME delivers long-term value to customers and stakeholders.
