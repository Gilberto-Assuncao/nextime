# NEXTIME --- USER STORIES

Version: 1.0 Status: Active Last Updated: 2026-07-21

# Purpose

This document defines the primary user stories for the NEXTIME MVP.

User stories describe functionality from the user's perspective and
guide sprint planning and implementation.

------------------------------------------------------------------------

# Authentication

## US-001

**As** a user

**I want** to sign in securely

**So that** I can access my company's workspace.

Acceptance Criteria:

-   Valid credentials allow access.
-   Invalid credentials show an appropriate error.
-   Sessions persist securely.

------------------------------------------------------------------------

# Companies

## US-002

**As** a company administrator

**I want** to create and manage company information

**So that** my organization is correctly configured.

Acceptance Criteria:

-   Company data can be created and edited.
-   Validation prevents invalid information.

------------------------------------------------------------------------

# Teams

## US-003

**As** an operations manager

**I want** to create teams

**So that** employees can be organized efficiently.

Acceptance Criteria:

-   Teams can be created, renamed and archived.
-   Team members can be assigned.

------------------------------------------------------------------------

# Workforce

## US-004

**As** a manager

**I want** to register employees

**So that** I can manage the workforce.

Acceptance Criteria:

-   Employee records include required information.
-   Records can be updated.

------------------------------------------------------------------------

# Projects

## US-005

**As** a manager

**I want** to create projects

**So that** time entries can be associated with work.

Acceptance Criteria:

-   Projects can be created and edited.
-   Teams can be linked to projects.

------------------------------------------------------------------------

# Time Tracking

## US-006

**As** a field employee

**I want** to register my working hours

**So that** my time is accurately recorded.

Acceptance Criteria:

-   Time entries are validated.
-   Entries are linked to the correct project.
-   Employees can view their own history.

------------------------------------------------------------------------

# Dashboard

## US-007

**As** a business owner

**I want** to view key metrics

**So that** I can monitor operations.

Acceptance Criteria:

-   Dashboard loads quickly.
-   Metrics are filtered by tenant.

------------------------------------------------------------------------

# Reports

## US-008

**As** a manager

**I want** to generate reports

**So that** I can analyze productivity.

Acceptance Criteria:

-   Reports support filters.
-   Export capability can be added in future releases.

------------------------------------------------------------------------

# Settings

## US-009

**As** an administrator

**I want** to configure system settings

**So that** the platform matches company needs.

Acceptance Criteria:

-   Only authorized users can modify settings.
-   Changes are persisted securely.

------------------------------------------------------------------------

# Goal

Provide a clear backlog of user-focused requirements that can be refined
into epics, tasks and sprint deliverables.
