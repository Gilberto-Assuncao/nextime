# NEXTIME --- MVP SCOPE

Version: 1.0 Status: Active Last Updated: 2026-07-21

# Purpose

This document defines the official scope of the Minimum Viable Product
(MVP) for the NEXTIME platform.

Its goal is to keep development focused on delivering a valuable first
release while preventing scope creep.

------------------------------------------------------------------------

# MVP Objectives

The MVP must:

-   Validate the product in the market.
-   Solve the core workforce time tracking problem.
-   Support real companies in production.
-   Provide a foundation for future expansion.

------------------------------------------------------------------------

# Included Features

## Authentication

-   User login
-   User logout
-   Session persistence
-   Protected routes

## Company Management

-   Create and manage companies
-   Company settings

## Team Management

-   Create teams
-   Assign employees

## Workforce

-   Employee registration
-   Employee profile management

## Projects

-   Create and manage projects
-   Assign teams

## Time Tracking

-   Manual time entry
-   Edit own entries
-   View history

## Dashboard

-   Operational overview
-   Basic KPIs

## Reports

-   Basic filtered reports

## Roles & Permissions

-   Owner
-   Administrator
-   Manager
-   Employee

------------------------------------------------------------------------

# Excluded from MVP

The following are intentionally postponed:

-   Payroll
-   Mobile offline mode
-   Public API
-   Marketplace integrations
-   Advanced analytics
-   AI features
-   White-label
-   Multi-company switching
-   Enterprise SSO
-   Advanced notifications

------------------------------------------------------------------------

# Success Criteria

The MVP is considered complete when:

-   Core workflows operate reliably.
-   Companies can use the platform daily.
-   Major defects are resolved.
-   Documentation is current.
-   Security requirements are satisfied.

------------------------------------------------------------------------

# Constraints

-   Small sprint deliveries.
-   No unnecessary complexity.
-   Maintain strict TypeScript.
-   Preserve clean architecture.
-   Avoid regressions.

------------------------------------------------------------------------

# Post-MVP Roadmap

Priority after MVP:

1.  Mobile application
2.  Advanced reporting
3.  Notifications
4.  Public API
5.  Integrations
6.  AI-assisted insights

------------------------------------------------------------------------

# Scope Control

New requests must be evaluated by:

-   Business value
-   Technical impact
-   Delivery effort
-   Alignment with product vision

Features outside the MVP require explicit prioritization.

------------------------------------------------------------------------

# Goal

Deliver a focused, production-ready MVP that validates the NEXTIME
product while establishing a solid foundation for future releases.
